# Lime 2.0 — роадмап «интересный ресурс с ВАУ-эффектом»

> Преемник [docs/PLAN.md](PLAN.md) (модернизация — выполнена) и внутреннего роадмапа Треков A/B
> (Трек A почти закрыт, Трек B B1–B3 закрыт). Утверждён 2026-06-10.
> Это документ-стратегия: этапы идут по порядку, внутри этапа пункты можно брать выборочно.

## Context

Заказчик хочет: (1) полную кастомизацию, а не «много блоков»; (2) ВАУ-эффект для ВСЕХ,
включая бесплатных — подписка не должна отрезать от красоты; (3) социальный слой —
делиться наработками, общаться, смотреть чужое; (4) взять лучшее у топовых продуктов
и **переосмыслить**, не нарушая авторских прав.

**Текущее состояние движка B** ([lime-doc.js](Lime_Editor/wwwroot/js/lime/lime-doc.js)):
9 типов блоков (heading, text, cover, cta, buttonGroup, stats, features, divider, spacer) —
**нет даже блока «картинка»**; плоский список блоков без вложенности; темы-токены CSS-vars;
компоненты; многостраничность через hash-роутинг; компиляция HTML на клиенте.

### Что берём у лидеров и как улучшаем (легально: реализуем концепции своим кодом — фичи не защищаются авторским правом, защищается код/дизайн/название, их не копируем)

| Продукт | Что любят пользователи | Наше переосмысление (и улучшение) |
|---|---|---|
| **Framer** | Лучшие анимации «из коробки», AI практичный а не гиммик | GSAP уже подключён → пресеты scrollytelling/микроинтеракций пер-блок, бесплатно всем |
| **Tilda** | Zero Block — свободный холст; жалоба №1: ломает адаптив, надо перевёрстывать под каждое устройство | Canvas-блок как ОДИН тип блока внутри нормального потока: свобода Tilda + наш адаптив по брейкпоинтам вокруг него |
| **Canva** | Brand Kit: «поменял цвет бренда — обновилось везде»; 250K шаблонов | Темы-токены уже есть → пресеты тем + Google Fonts + палитра из логотипа; шаблоны добирает сообщество |
| **Figma Community** | Шаринг и remix чужих наработок двигает рост (+400K юзеров с одной кампании) | **Remix**: JSON-модель делает «клонируй чужой сайт/секцию к себе» тривиальным — это наш главный соц-механизм |
| **Framer Marketplace** | Креаторы публикуют шаблоны, остаются с аудиторией | Галерея сообщества: сначала бесплатный обмен, платный маркетплейс — потом |
| **Webflow** | SEO/мета-инструменты | Per-page SEO после реальных роутов страниц |
| Тренды 2026 | Glassmorphism, aurora, dark mode по умолчанию, scrollytelling | Пресеты тем («Стекло», «Аврора», «Тьма») и эффектов — один клик |

**Правовая рамка remix:** ремиксится только контент наших пользователей и только с их
согласия — чекбокс «Разрешить использовать как шаблон» при публикации в галерею + плашка
атрибуции «by X · remixed by Y». Системные шаблоны — наши собственные.

## Этап 0 — Фундамент (архитектурные долги, БЕЗ этого остальное встанет)

Из критического разбора 2026-06-10; порядок важен.

- **0.1 `children[]` в схему блока** — пока документов ноль. Рендерер рекурсивный; редактор может пока не давать вкладывать. Разблокирует 1.1.
- **0.2 Серверная компиляция из JSON** — гонять lime-doc.js через Node как publish-step (`node render-cli.js`); Publish = пересборка из JSON на сервере; команда «republish all». Лечит «замороженный HTML», разблокирует 0.3, 3.x (превью галереи).
- **0.3 Реальные роуты страниц** `/u/{user}/{slug}/{page}` вместо hash на публикации (hash остаётся как превью в редакторе). Per-page `<title>`/мета. Разблокирует SEO.
- **0.4 Undo/redo + конфликт автосейва** — стек снапшотов JSON-документа (дёшево, ради этого JSON и делался); optimistic concurrency по `UpdatedAt`.
- **0.5 Медиа-блоки в движке B**: image / gallery / video(embed) + интеграция готового media-picker (`MediaController` уже есть). Без картинок «кастомизация» — пустой звук.
- **0.6 Отдельный домен для публикаций + CSP** — user-content нельзя держать на origin с Identity-cookie (stored-XSS). Caddy уже стоит. Блокер перед публичным деплоем, не перед разработкой.
- **0.7 Заморозка legacy-редактора** (constructor.js — только багфиксы) + импортёр блоб→JSON (кнопка «перенести сайт на новый движок»).

## Этап 1 — Кастомизация без потолка

- **1.1 Контейнеры и колонки** (на базе 0.1): блоки section/columns/card, drag-and-drop блоков внутрь контейнеров и между ними (drag-reorder из бэклога — сюда же). Любой «готовый» блок можно разобрать на примитивы.
- **1.2 Глубокий инспектор** (декларативный реестр контролов): градиенты, многослойные тени, hover-состояния, border-radius, blend/opacity, полная типографика (Google Fonts picker), фон-картинка с overlay. Все значения — в `styles[base|tablet|mobile]`, адаптив не ломается.
- **1.3 Темы 2.0 / Brand Kit**: 8–10 профессиональных пресетов («Аврора», «Стекло», «Тьма», «Минимал»...); смена темы = мгновенный рестайл всего сайта (CSS-vars уже умеют); палитра из загруженного логотипа (color extraction на клиенте, без LLM).
- **1.4 Canvas-блок (наш «Zero Block, который не ломает адаптив»)**: внутри одного блока — свободное позиционирование элементов (x,y,% от контейнера), на tablet/mobile — свои раскладки ИЛИ авто-стек фолбэк. Самый сложный пункт этапа — делать последним.
- **1.5 Анимации из коробки**: пресеты появления/parallax/scrollytelling в инспекторе (lime-animate.js + GSAP), микроинтеракции hover для кнопок/карточек. `prefers-reduced-motion` обязателен.

## Этап 2 — AI и ВАУ для всех

- **2.1 AI всем с квотой** (архитектура из A2 остаётся: Haiku 4.5, structured output → блоки, серверная квота). Бесплатным — N генераций/мес; подписка = больше квота + умнее модель. **Красота (блоки, темы, анимации, сообщество) — бесплатна всем; платим за объём, не за доступ.**
- **2.2 «Сайт за 60 секунд» как онбординг**: поле промпта прямо на лендинге, генерация превью ДО регистрации, регистрация — чтобы сохранить. Главный wow-момент + воронка.
- **2.3 ✨-инструменты в редакторе**: переписать/сократить/тон текста по выделению; AI-подбор темы под описание бизнеса.
- **2.4 Магия первой публикации**: конфетти, красивая карточка ссылки, автогенерация OG-превью (скриншот через серверный рендер из 0.2), QR-код сайта.
- **2.5 Экспорт в код — «твой сайт принадлежит тебе»** (анти-vendor-lock-in, дифференциатор: Tilda/Framer наружу чистый код не отдают):
  - **Tier 1 — статический ZIP**: `renderSite(json)` из 0.2 + медиа-файлы → архив `index.html` + assets, открывается с диска / кидается на любой хостинг. Просто, делается сразу после 0.2.
  - **Tier 2 — React-проект (Vite)**: кодогенератор JSON-док → готовый пакет: `package.json` + vite config, `src/blocks/*.jsx` (по компоненту на тип блока), `src/pages/*` + react-router (страницы из doc.pages), тема → CSS-переменные, контент → props/`content.json`, GSAP-анимации → npm-зависимость. Front-only: формы экспортируются с настраиваемым webhook-URL (env) или заглушкой. `npm i && npm run dev` работает из коробки — golden-тест в CI.
  - **Tier 3 (по спросу)**: Vue / Astro / Next static — тот же генератор, другие таргеты-шаблоны.

## Этап 3 — Сообщество (соцсеть вокруг наработок)

- **3.1 Галерея**: opt-in публикация в витрину (чекбокс + лицензия remix), превью-скриншоты (из 0.2), лайки, просмотры, теги/категории.
- **3.2 Публичный профиль автора** (`/u/{user}` уже существует как префикс): аватар, био, сайты автора, суммарные лайки.
- **3.3 Remix** (ядро соц-механики): «Использовать как шаблон» — клон JSON-документа к себе одним кликом; то же для отдельных секций («поделиться наработкой»). Атрибуция автора. Это превращает каждого пользователя в поставщика шаблонов → решает «у Tilda 550 блоков, у нас 9».
- **3.4 Комментарии к сайтам в галерее + подписка на автора** (уведомления в кабинете, без email-спама на старте).
- **3.5 Модерация**: жалоба на сайт/коммент, скрытие из галереи, инструменты в существующем `AdminController`. Обязательна ДО открытия галереи.
- **3.6 Отложено сознательно**: лента, личные сообщения, платный маркетплейс (creators %), real-time коллаборация. Делаем только когда 3.1–3.4 покажут активность — иначе соцсеть-призрак.

## Этап 4 — За рамками (задел в архитектуре, не делаем)

Биллинг (нужен к моменту, когда AI-квоты начнут упираться), кастомные домены,
A/B-тесты, командная работа, экспорт-таргеты Tier 3 (Vue/Astro/Next).

## Порядок и грубые оценки

| Этап | Объём | Комментарий |
|---|---|---|
| 0.1–0.5 | ~2-3 нед | 0.1 — часы; 0.2 — главный риск этапа |
| 0.6–0.7 | ~1 нед | 0.6 можно отложить до деплоя |
| 1 (без 1.4) | ~3-4 нед | 1.2 и 1.5 дают видимый результат быстро |
| 1.4 canvas | ~2-3 нед | отдельно, после остального |
| 2 (AI+wow) | ~2-3 нед | архитектура AI уже спроектирована в старом роадмапе |
| 2.5 экспорт | ZIP ~3-5 дн; React ~1.5-2 нед | ZIP — сразу после 0.2; React-генератор — отдельный модуль, не трогает редактор |
| 3.1–3.5 | ~3-4 нед | новые таблицы: GalleryEntry/Like/Comment/Follow/Report |

Рекомендуемый старт: **0.1 → 0.5 (картинки!) → 0.2 → 1.2 → 1.3** — уже после этого
продукт визуально другой. AI (этап 2) можно вклинить раньше этапа 1, если хочется wow быстрее.

## Verification

- node-self-test рендерера расширять на каждый новый тип блока/children (паттерн `B2-RENDER-OK` уже есть).
- Скилл `lime-ui-tester` (Playwright): smoke редактора B, drag-and-drop, темы, публикация, галерея, remix-флоу.
- Этап 0.2: `renderSite(json)` на сервере байт-в-байт совпадает с клиентской компиляцией (golden-тест).
- Этап 2.5: экспортированный ZIP открывается с диска с рабочими стилями/медиа; React-пакет — `npm i && npm run build` проходит в CI на чистой машине (golden-тест), страницы и адаптив совпадают с опубликованной версией.
- Этап 3: чужой непубличный сайт не виден в галерее; remix без галочки-разрешения невозможен (тест ownership в Lime.Tests).
- Безопасность: published-домен не получает Identity-cookie; CSP на публичных страницах.
